# Frontend UI Engineer Skill

## What I do
I implement beautiful, responsive, and accessible frontend interfaces using modern web technologies including Astro, React, Tailwind CSS, and shadcn/ui design principles.

## When to use me
- When building new UI components or pages
- When implementing responsive layouts for mobile and desktop
- When creating reusable design system components
- When optimizing frontend performance and accessibility
- When implementing dark/light mode themes
- When creating interactive form elements and controls

## Required checks
- Ensure responsive design works on mobile (375px), tablet (768px), and desktop (1024px+)
- Verify color contrast meets WCAG AA standards
- Test keyboard navigation and focus states
- Validate HTML semantics and ARIA attributes
- Check for consistent spacing and typography
- Ensure smooth transitions and animations

## Output format
- Clean, well-structured Astro/React components
- Tailwind CSS utility classes with consistent design tokens
- Proper TypeScript types for component props
- Responsive layouts with mobile-first approach
- Accessible markup with proper ARIA labels

## Failure handling
- If components break existing layouts, revert changes and document issues
- If accessibility violations found, prioritize fixes before proceeding
- If performance degrades, optimize images and lazy load components
- If dark mode inconsistencies occur, check CSS variable definitions

## Design principles
- Modern SaaS/developer tools aesthetic
- Clear visual hierarchy with proper spacing
- Consistent color palette with primary, secondary, and accent colors
- Subtle shadows and borders for depth
- Smooth hover/focus/active states
- Professional typography with readable font sizes
- Cards with rounded corners and subtle borders
- Gradient backgrounds for visual interest
- Iconography for better visual communication

## Component patterns
- Use Astro components for static content
- Use React for interactive components
- Implement proper loading states
- Add error boundaries and fallback UI
- Create empty states with helpful guidance
- Include success/error feedback for user actions

## Performance considerations
- Optimize images with proper sizing and formats
- Implement lazy loading for below-the-fold content
- Use code splitting for large bundles
- Minimize CSS with Tailwind purging
- Preload critical resources

## Accessibility requirements
- Proper heading hierarchy (h1-h6)
- Alt text for all images
- ARIA labels for interactive elements
- Keyboard navigable interface
- Focus visible states
- Screen reader friendly content
- Color contrast ratio ≥ 4.5:1 for normal text
