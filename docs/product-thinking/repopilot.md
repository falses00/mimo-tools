# RepoPilot - 苏格拉底式产品自问

## 1. RepoPilot 解决谁的问题？

RepoPilot 的目标用户是：
- **开发者**：想找到有趣的开源项目学习或贡献
- **技术负责人**：想评估引入哪些开源依赖
- **安全工程师**：想审查开源项目的安全性
- **DevOps 工程师**：想快速部署开源项目

这些用户的共同痛点：**找项目难、评估难、部署难**。

## 2. 用户为什么不用 GitHub Trending 自己找？

GitHub Trending 的问题：
1. **信息过载**：每天几百个项目，不知道看哪个
2. **没有安全审查**：不知道项目是否安全
3. **没有部署指南**：不知道怎么本地运行
4. **没有依赖分析**：不知道项目依赖是否安全
5. **没有评分**：不知道项目质量如何

RepoPilot 解决这些问题：**发现 → 审查 → 部署，一站式解决**。

## 3. 用户为什么需要安全审查？

开源项目的安全风险：
1. **恶意代码**：postinstall 脚本可能执行恶意命令
2. **依赖漏洞**：依赖的包可能有已知漏洞
3. **数据泄露**：代码可能收集用户数据
4. **供应链攻击**：维护者账号可能被入侵

## 4. 用户为什么需要本地部署管家？

本地部署的痛点：
1. **不知道怎么 clone**
2. **不知道怎么安装依赖**
3. **不知道怎么配置环境变量**
4. **不知道怎么启动**
5. **不知道怎么停止**
6. **不知道端口冲突怎么办**

## 5. 哪些步骤必须人工确认？

必须人工确认：
1. **选择部署目录** - 不能默认写 C 盘
2. **确认 clone** - 不能自动 clone 未知仓库
3. **确认 install** - 不能自动执行未知脚本
4. **确认 start** - 不能自动启动服务
5. **配置 API key** - 不能自动读取用户密钥

## 6. 哪些命令绝不能自动执行？

绝不能自动执行：
- `curl | sh`
- `wget | sh`
- `sudo`
- `rm -rf /`
- `chmod 777`
- `eval`
- `base64 -d`
- `powershell`
- `Invoke-WebRequest`
- postinstall/preinstall 脚本（除非用户确认）

## 7. 如何避免恶意仓库风险？

检测策略：
1. **检查 package.json scripts** - 检测危险关键词
2. **检查 dependencies** - 检测已知漏洞包
3. **检查 Dockerfile** - 检测危险命令
4. **检查 GitHub Actions** - 检测危险步骤
5. **检查 README** - 检测可疑链接
6. **检查 Stars/Forks** - 判断项目可信度

## 8. 如何判断 package.json scripts 是否危险？

危险关键词：
- postinstall
- preinstall
- curl
- wget
- sudo
- rm -rf
- chmod
- eval
- base64
- powershell

## 9. 如何判断 Dockerfile 是否危险？

危险命令：
- ADD/COPY 可疑文件
- RUN curl | sh
- RUN sudo
- EXPOSE 敏感端口
- ENV 传递密钥

## 10. 如何判断 GitHub Actions 是否危险？

危险步骤：
- 使用不可信 action
- 传递 secrets
- 执行 shell 脚本
- 部署到生产环境
