# LifePilot - 苏格拉底式产品自问

## 1. 这个项目真正解决谁的问题？
**目标用户**：忙碌的职场人士、自由职业者、学生、创业者
**核心痛点**：
- 每天有很多事情要做，但没有时间整理
- 想记录生活，但传统待办 App 太繁琐
- 需要一个"一句话就能用"的生活助手

## 2. 用户第一次打开页面时，是否知道怎么用？
**是** - 页面有明确的新手引导和示例输入
用户看到示例："我今天完成了博客首页优化，跑步 3 公里，明天上午要投简历"
就能理解如何使用

## 3. 用户输入什么？
**一句话自然语言**，例如：
- "我今天写完了博客，跑了 3 公里，明天要投简历"
- "交了房租 3000，买了服务器 50 元，周五前完成项目部署"
- "今天健身 1 小时，阅读 30 页，明天 9 点开会"

## 4. 系统输出什么？
**结构化生活记录**：
- 已完成事项
- 待办事项
- 提醒事项
- 习惯记录
- 账单/支出
- 项目进度
- 智能建议

## 5. 输出结果能否直接行动？
**是** - 每个待办都有优先级和截止时间
用户可以直接执行，不需要二次整理

## 6. 这个项目和普通 demo 的区别是什么？
**完整业务闭环**：
- 输入 → 解析 → 编辑 → 保存 → 历史 → 导出
- 普通 demo 只有输入 → 输出

## 7. 它有没有完整业务闭环？
**是**：
1. 用户输入一句话
2. 系统解析成结构化数据
3. 用户可编辑、删除、标记完成
4. 保存到 localStorage 或数据库
5. 查看历史记录
6. 导出 Markdown 日报
7. 设置个人偏好

## 8. 它是否需要后端？
**可选**：
- 无后端：localStorage fallback
- 有后端：数据同步、多设备、AI 增强

## 9. 它是否需要数据库？
**可选**：
- 无数据库：localStorage
- 有数据库：Supabase 持久化

## 10. 它是否需要 RAG？
**不需要** - 当前版本使用规则解析
未来可选：AI 增强解析、个性化建议

## 11. 没有 API key 时还能不能用？
**能** - 本地规则解析 fallback

## 12. 没有数据库时如何 fallback？
**localStorage** - 数据保存在浏览器

## 13. 用户怎么保存历史？
- 点击"保存"按钮
- 自动保存到 localStorage
- API Mode 下保存到后端

## 14. 用户怎么编辑结果？
- 点击编辑按钮
- 修改标题、分类、日期、优先级
- 保存修改

## 15. 用户怎么撤销误操作？
- 删除前确认提示
- 历史记录可查看
- localStorage 有版本

## 16. 用户怎么导出报告？
- 点击"导出日报"按钮
- 生成 Markdown 格式
- 复制到剪贴板或下载

## 17. 用户下一步该做什么？
- 查看今日时间线
- 执行待办事项
- 设置提醒
- 查看明日计划

## 18. 面试官看到它，会觉得技术点在哪里？
- 自然语言处理（规则解析）
- 前端架构（组件化、状态管理）
- 后端 API 设计
- 数据库设计
- 全栈能力
- 产品设计能力

## 19. 它和同类产品相比有什么差异？
**差异化**：
- 一句话输入，零学习成本
- 自动分类（已完成/待办/习惯/账单）
- 中文优先
- 开源免费
- 全栈能力展示

## 20. 当前实现是否仍然像 demo？
**否** - 当前实现包含：
- 完整工作台
- 编辑/删除/完成
- 保存/恢复
- 导出
- 设置
- FAQ
- 中文化
