# InterviewPilot - 苏格拉底式产品自问

## 1. 这个项目真正解决谁的问题？

InterviewPilot 的目标用户是正在求职、准备面试的人。具体包括：

- **应届毕业生**：第一次参加面试，不知道如何准备
- **跳槽者**：需要针对新岗位准备面试
- **转行者**：需要学习新领域的面试技巧
- **面试官**：需要准备面试问题

这些用户的共同痛点是：**不知道面试会问什么，不知道如何回答**。

## 2. 用户为什么不用普通面试准备 App？

普通面试准备 App 的问题：

1. **问题库太泛**：覆盖所有行业，但不针对具体岗位
2. **没有个性化**：不根据用户简历和 JD 定制
3. **没有评分**：不知道回答好不好
4. **没有 STAR 框架**：没有结构化回答指导
5. **没有复盘**：没有历史记录和改进分析

InterviewPilot 解决这些问题：**根据简历和 JD 生成个性化面试问题，评分回答，提供 STAR 建议**。

## 3. 用户输入什么？

用户输入：
- **简历文本**：粘贴或上传简历
- **岗位 JD**：粘贴岗位描述

系统自动分析：
- 技能匹配度
- 缺口技能
- 面试问题
- STAR 回答建议

## 4. 系统输出什么？

系统输出：
- **匹配分数**：简历和 JD 的匹配程度
- **技能覆盖**：简历中有哪些技能匹配 JD
- **缺口技能**：JD 需要但简历没有的技能
- **简历改进建议**：如何优化简历
- **面试问题**：根据简历和 JD 生成的面试问题
- **STAR 回答建议**：每个问题的结构化回答建议

## 5. 输出结果能否直接行动？

**是** - 用户可以：
- 根据匹配分数调整简历
- 根据缺口技能学习新技能
- 根据面试问题准备回答
- 根据 STAR 建议练习回答

## 6. 这个项目和普通 demo 的区别是什么？

**完整业务闭环**：
- 输入简历和 JD → 分析匹配度 → 生成面试问题 → 提供 STAR 建议 → 保存历史 → 导出报告

**普通 demo**：
- 输入 → 输出（无保存、无历史、无导出）

## 7. 它有没有完整业务闭环？

**是**：
1. 用户输入简历和 JD
2. 系统分析匹配度
3. 系统生成面试问题
4. 用户查看 STAR 建议
5. 用户保存分析结果
6. 用户导出面试准备报告
7. 用户查看历史记录

## 8. 它是否需要后端？

**可选**：
- 无后端：本地规则分析
- 有后端：AI 增强分析、数据同步

## 9. 它是否需要数据库？

**可选**：
- 无数据库：localStorage
- 有数据库：Supabase 持久化

## 10. 它是否需要 RAG？

**不需要** - 当前版本使用规则分析
未来可选：AI 增强分析、行业知识库

## 11. 没有 API key 时还能不能用？

**能** - 本地规则分析 fallback

## 12. 没有数据库时如何 fallback？

**localStorage** - 数据保存在浏览器

## 13. API Mode 和 Static Demo Mode 的区别是什么？

| 维度 | Static Demo Mode | API Mode |
|------|------------------|----------|
| 分析 | 本地规则 | 后端 API |
| 准确性 | 基础匹配 | 可扩展 AI |
| 保存 | localStorage | API + 数据库 |
| 离线 | 完全可用 | 需要网络 |

## 14. 数据保存在哪里？

| 模式 | 保存位置 |
|------|----------|
| 本地模式 | localStorage |
| API Mode | API + 数据库 |

## 15. 隐私如何保证？

1. **本地优先**：默认不上传数据
2. **用户控制**：只有点击"保存"才存储
3. **透明提示**：明确告知数据保存位置
4. **无追踪**：不收集用户行为数据

## 16. 这个项目面试时展示什么技术点？

1. **NLP 处理**：关键词提取、技能匹配
2. **前端架构**：组件化、状态管理
3. **后端 API**：RESTful 设计
4. **数据持久化**：localStorage、API、数据库
5. **产品设计**：用户旅程、交互设计

## 17. 它为什么不是 demo？

| 维度 | Demo | InterviewPilot |
|------|------|----------------|
| 输入 | 固定示例 | 简历 + JD |
| 输出 | 静态文本 | 匹配分数、面试问题、STAR 建议 |
| 保存 | 无 | localStorage + API |
| 历史 | 无 | 有 |
| 导出 | 无 | Markdown |
| 个性化 | 无 | 根据简历和 JD |

## 18. 它还有哪些未完成的高级能力？

1. **AI 增强分析**：接入 LLM 提高准确率
2. **模拟面试**：AI 面试官提问
3. **语音回答**：语音转文字
4. **回答评分**：AI 评分回答质量
5. **行业知识库**：特定行业面试问题
6. **面试录音**：记录面试过程

## 19. 下一阶段如何接入真正 LLM？

**方案**：
1. 前端发送简历和 JD 到 API
2. API 调用 MiMo-V2.5-Pro
3. LLM 返回分析结果
4. 前端展示结果

## 20. 下一阶段如何接入 Supabase？

**方案**：
1. 创建 Supabase 项目
2. 运行 migration
3. 配置环境变量
4. API 保存到 Supabase

## 用户旅程

1. **首次访问**：看到产品介绍和示例
2. **输入简历**：粘贴或上传简历
3. **输入 JD**：粘贴岗位描述
4. **点击分析**：等待分析
5. **查看结果**：匹配分数、技能覆盖、缺口技能
6. **查看面试问题**：根据简历和 JD 生成
7. **查看 STAR 建议**：每个问题的结构化回答
8. **保存结果**：保存到 localStorage
9. **导出报告**：生成 Markdown 面试准备报告

## 失败场景和 Fallback 策略

| 场景 | 失败原因 | Fallback |
|------|----------|----------|
| API 不可用 | 网络问题/服务器错误 | 本地规则分析 |
| 分析不准确 | 简历/JD 格式问题 | 用户手动编辑 |
| 保存失败 | localStorage 满 | 提示清理空间 |
| 导出失败 | 浏览器不支持 | 复制到剪贴板 |
